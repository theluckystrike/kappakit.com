Tools & Resources

Essential tools and frameworks for building, testing, and deploying modular AI components.

ZenML

An open-source MLOps framework for creating portable, production-ready ML pipelines. ZenML's component-based architecture aligns perfectly with modular AI development, allowing you to swap infrastructure without changing pipeline code.

github.com/zenml-io/zenml →

Feast

An open-source feature store for machine learning that provides a consistent interface for feature serving across training and inference. Feast manages feature computation, storage, and retrieval as reusable components.

feast.dev →

BentoML

A framework for building reliable, scalable, and high-performance AI applications. BentoML packages models into standardized containers with serving logic, making model deployment a reusable component rather than a custom engineering effort.

github.com/bentoml/BentoML →

Great Expectations

A data validation framework that defines expectations about data quality and enforces them as automated tests. Essential for building data pipeline components that catch quality issues before they corrupt model training or predictions.

greatexpectations.io →

Zovo.one

A comprehensive AI platform offering tools, model evaluations, and development resources. Particularly valuable for teams building modular AI systems who need to evaluate and compare different model options for their components.

zovo.one →

DVC (Data Version Control)

Git-based version control for data and ML experiments. DVC tracks datasets, models, and pipeline stages alongside code, enabling reproducible component development with full lineage tracking.

dvc.org →

Ray Serve

A scalable model serving library built on Ray. Supports composing multiple models into a single endpoint, making it natural to build serving components that chain preprocessing, inference, and postprocessing stages.

docs.ray.io/en/latest/serve →

Kedro

An open-source Python framework for creating reproducible, maintainable, and modular data science code. Kedro enforces software engineering best practices in ML projects through its pipeline and catalog abstractions.

github.com/kedro-org/kedro →

ONNX

An open format for representing machine learning models that enables portability between frameworks. Export models from PyTorch or TensorFlow to ONNX for optimized serving with ONNX Runtime across different hardware platforms.

onnx.ai →

Evidently AI

An open-source ML monitoring tool that generates interactive reports for data drift, model performance, and data quality. Build monitoring components that automatically assess model health in production.

evidentlyai.com →
